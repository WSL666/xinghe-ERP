## 属性数据库（schema 2）使用说明

本目录是构建工具与产物。

## 文件
- `build_attr_db.py` —— 更新脚本（全量重建，幂等）
- `attr_new_db.json` —— 本次生成的数据库
- `build_report.txt` —— 构建报告（规模 + 体检）

## 以后更新数据库
把新的 output 文件夹准备好后，一条命令即可（默认输入/输出已写死在脚本顶部）：

```
python build_attr_db.py
```
换输入/输出路径：
```
python build_attr_db.py --input "X:\新output" --output "Y:\attr_db.json"
```
脚本会在数据库同目录生成 `build_report.txt`。每次都是全量重建，重跑结果逐字节一致。

## 数据库结构（6 张表）

| 表 | 键 -> 值 | 用途 |
|----|----------|------|
| `refPid` | refPid -> pid | **查 pid 的主键**，1:1 无冲突 |
| `names` | propName -> pid | refPid 缺失时兜底 |
| `values` | `pid\|propValue` -> vid | pid 命中后查 vid |
| `templates` | templateId -> {pid: templatePid} | **精确** templatePid（按类目） |
| `defaultTemplatePid` | pid -> templatePid | 未知类目时的兜底（频次最高） |
| `props` | name -> {pid, templatePid, source} | 旧版兼容层 |

## 匹配算法（三步，每步有兜底）
```
# 1) pid：优先 refPid，缺失再用名字
pid = refPid[refPid] or names[propName]
# 2) vid
vid = values[pid + '|' + propValue]
# 3) templatePid：知道类目模板就精确，否则兜底
tpid = (templates[templateId][pid]) or defaultTemplatePid[pid]
```

## 如何发挥最大精度（升级 popup.js）
现有 popup.js 直接用 `props`/`values` 也能跑（兼容层），但要用上 refPid 和 templates 精确匹配，
需把 popup.js 里「从本地属性数据库补全 pid/vid/templatePid」那段换成下面的逻辑。
（关键点：页面采集到的 `p.refPid` 现在终于被用上了。）

```js
const attrDB = await dbResp.json();
const refPidDB     = attrDB.refPid || {};
const namesDB      = attrDB.names  || {};
const valuesDB     = attrDB.values || {};
const templatesDB  = attrDB.templates || {};
const defTpidDB    = attrDB.defaultTemplatePid || {};
const pageTemplateId = String(collectedData.templateId || '');  // 若以后采集到类目模板

enrichedProps = filteredProps.map(p => {
  const pn = (p.propName || '').trim();
  const pv = (p.propValue || '').trim();
  const ref = String(p.refPid || '').trim();
  // 1) pid：refPid 主键优先
  let pid = (ref && refPidDB[ref]) ? String(refPidDB[ref])
          : (namesDB[pn] ? String(namesDB[pn])
          : (attrDB.props && attrDB.props[pn] ? String(attrDB.props[pn].pid) : ''));
  pid = pid || p.pid || '';
  // 2) templatePid：精确 -> 兜底
  let tpid = '';
  if (pageTemplateId && pid && templatesDB[pageTemplateId] && templatesDB[pageTemplateId][pid])
    tpid = String(templatesDB[pageTemplateId][pid]);
  else if (pid && defTpidDB[pid]) tpid = String(defTpidDB[pid]);
  else tpid = p.templatePid || '';
  // 3) vid
  let vid = p.vid || '';
  if (!vid && pv && pid) {
    const vk = pid + '|' + pv;
    const vm = valuesDB[vk];
    if (vm != null) vid = (typeof vm === 'string') ? vm : (vm.vid || '');
    else {
      for (const part of pv.split('、')) {
        const m = valuesDB[pid + '|' + part.trim()];
        if (m != null) { vid = (typeof m === 'string') ? m : (m.vid || ''); break; }
      }
    }
  }
  return { propName:p.propName, refPid:p.refPid||'', pid, templatePid:tpid,
           numberInputValue:p.numberInputValue||'', valueUnit:p.valueUnit||'',
           vid, propValue:p.propValue };
});
```
> 替换范围：popup.js 中 `// ===== 从本地属性数据库补全 pid/vid/templatePid =====` 到
> `const propsJson = JSON.stringify(...)` 之间的 `filteredProps.map(...)` 主体。其余完全不动。

## 体检结论（见 build_report.txt）
- refPid->pid 冲突：**0**（主键完全干净）
- (templateId,pid)->templatePid 冲突：23（极少数模板版本差异，已按频次取定）
- propName->pid 真冲突：3（填充材料/主体材质/里衬成分，靠 refPid 即可避开）
- pid|value->vid 多值：255（手机型号等，已取频次最高）
