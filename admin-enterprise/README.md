# admin-enterprise · 企业管理后台（2级）

> **状态：占位预留，当前未实现。** 等 1级平台超管系统（`admin-platform/`）搭好后再开发。

## 这是什么

2级企业管理员后台。企业管理员（owner / admin）登录后，只能管理**本企业内部**的成员和数据，**不能跨企业**，更看不到全平台数据。

## 三级权限体系

| 级别 | 角色 | 系统 | 数据可见范围 |
|------|------|------|------------|
| **1级** | 平台超管 | `admin-platform/` | 全平台所有人 |
| **2级** | 企业管理员 | `admin-enterprise/`（本目录） | 仅本企业 |
| **3级** | 普通用户 | `backend/` + `frontend/` | 仅自己 |

数据可见性逐级收窄：1级看所有，2级看本企业，3级看自己。三级**登录态完全隔离**，互不共用。

## 未来职责

- 查看本企业成员清单与各自用量
- 邀请/移除成员，调整成员角色（admin / member）
- 查看本企业任务记录、生成图、导出历史
- 查看本企业金豆余额与消费明细
- 本企业级别的统计看板

## 数据来源

复用主应用已有的多租户表，无需新建核心表：

- `enterprises` —— 企业信息
- `enterprise_members` —— 成员关系（含 role）
- `imports` —— 任务记录（按 enterprise 过滤）
- `bean_transactions` —— 金豆流水（按 enterprise 过滤）
- `users` —— 用户信息（仅本企业成员）

所有查询都以 `enterprise_id` 为隔离边界，**绝不能**裸查全表。
