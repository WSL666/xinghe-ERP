# Temu 商品属性数据库维护说明

## 文件位置

```
E:\workplace\temu-collector\attr_db.json
```

## 数据库结构

```json
{
  "props": {
    "材料":    {"pid": 89,   "templatePid": 913001, "source": "xlsx"},
    "供电方式": {"pid": 1425, "templatePid": 903470, "source": "api"},
    "颜色":    {"pid": 63,   "templatePid": 903470, "source": "xlsx"}
  },
  "values": {
    "89|不锈钢":   {"vid": "2171", "source": "xlsx"},
    "1425|USB供电": {"vid": "36791", "source": "api"},
    "63|黑色":     {"vid": "100", "source": "xlsx"}
  }
}
```

### source 字段说明

| source 值 | 含义 | 优先级 | 合并规则 |
|-----------|------|:------:|----------|
| `api` | 卖家后台模板API直接获取 | **最高** | 覆盖已有值（即使冲突） |
| `xlsx` | 从导出xlsx的产品属性列提取 | 普通 | 仅当数据库中不存在该key时才追加 |

### 数据映射关系

| 索引 | Key | Value | 映射关系 |
|------|-----|-------|----------|
| `props` | propName（属性名） | `{pid, templatePid, source}` | **propName ↔ pid** 全局一对一固定 |
| `values` | `"pid\|propValue"`（属性ID+值） | `{vid, source}` | **(pid, propValue) ↔ vid** 跨属性不重复 |

## 8 字段说明

以 `{"propName":"电池属性","refPid":2153,"pid":1539,"templatePid":935818,"numberInputValue":"","valueUnit":"","vid":"52029","propValue":"可充电电池"}` 为例：

| 字段 | 示例值 | 来源 | 说明 |
|------|--------|:--:|------|
| **propName** | 电池属性 | 页面 | 属性名称（人类可读） |
| **propValue** | 可充电电池 | 页面 | 属性值（人类可读） |
| **refPid** | 2153 | 页面自带 | 引用属性ID，指向父级属性的pid |
| **pid** | 1539 | 数据库 | 属性ID，全局唯一，需从卖家后台模板获取 |
| **vid** | 52029 | 数据库 | 属性值ID，全局唯一，需从卖家后台模板获取 |
| **templatePid** | 935818 | 数据库 | 类目模板ID，同类目下所有属性共用 |
| **numberInputValue** | "" | 页面 | 数值型属性时存数字（如电池容量300） |
| **valueUnit** | "" | 页面 | 值的单位（如mAh、cm、g） |

### 映射关系

```
propName  ←→  pid         一对一，全局固定。"电池属性"永远是 pid=1539
propValue ←→  vid         一对一（在同一个pid下）。"可充电电池"永远是 vid=52029
refPid    →   另一个pid    依赖链，子属性指向父属性。如"可充电电池"的 refPid 指向"电池属性"的 pid
templatePid               类目模板容器，同一类目下所有属性共用
```

### 字段顺序（JSON.stringify 锁定）

```javascript
['propName','refPid','pid','templatePid','numberInputValue','valueUnit','vid','propValue']
```

## 数据来源

数据库从 **卖家后台导出的 xlsx** 文件中提取。这些 xlsx 的「产品属性」列包含完整的 8 字段 JSON，其中 pid/vid/templatePid 已经由卖家后台自动填充。

## 如何扩充数据库

### 步骤 1：准备数据

**方式A：xlsx 文件**（source=xlsx，低优先级）

将新的 xlsx 文件放到一个文件夹中（可以有子文件夹）。要求：
- 必须是 `.xlsx` 格式
- 59 列表头与模板一致（第一列为「产品标题」）
- 「产品属性」列（第29列）包含完整的 8 字段 JSON
- pid/vid/templatePid 有实际值（非 0/空）

**方式B：模板 API JSON**（source=api，高优先级，覆盖已有值）

从卖家后台 F12 → Network → 找到 `template/query` 请求的 Response，复制完整 JSON。

### 步骤 2：把数据路径和本说明文档发给 Claude

在新对话中说：

> 这是属性数据库扩充说明文档，我需要扩充数据库：
> - 新 xlsx 数据在 `[文件夹路径]`（可选）
> - 模板 API JSON 数据如下/在文件中（可选）
> 请帮我更新 `E:\workplace\temu-collector\attr_db.json`

### 步骤 3：Claude 会自动

**xlsx 文件** → 遍历文件夹中所有 xlsx → 提取「产品属性」列 JSON → 提取 propName→pid、pid|propValue→vid → 去重后以 source=xlsx 合并

**API JSON 文件** → 读取 JSON → 从 result.properties 中提取 name/pid/templatePid（props）和 values 中的 value/vid（values）→ 以 source=api 覆盖已有值

最后报告：新增了多少 props、values，覆盖了多少旧值

## 插件逻辑

插件导出 xlsx 时的数据流：

```
采集竞品页面 → goodsProperty（只有 propName/propValue/refPid）
       ↓
查询 attr_db.json:
  propName → props[propName] → pid, templatePid
  pid|propValue → values[pid|propValue] → vid
       ↓
数据库命中 → 填入 pid/vid/templatePid
数据库未命中 → 留空
       ↓
过滤「商品编号」「产地」
       ↓
JSON.stringify（replacer数组锁定字段顺序）
       ↓
写入 xlsx 第29列「产品属性」
```

## 注意事项

- **propValue 不唯一**：同一个值在不同属性下有不同的 vid。因此 values 的 key 是 `"pid|propValue"` 组合，不是单独的 propValue
- **API 权威覆盖**：API 来源的 propName 映射会覆盖 xlsx 来源的旧值，确保 pid 正确
- **数据库只增不减**：xlsx 来源只追加不覆盖；API 来源覆盖已有值
